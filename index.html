Boston Police Index: Feb 2023-May 2023

Tools Utilized: TypeScript React, Taiga, Scrum Cycle, PostgresSQL

Police discrimination is a serious issue that affects everywhere in the United States. A site called Woke Windows provided information about officers and their activites in Boston. This was so that lawyers and advocates could easily access data that could show that certain officers were treating a group of people in a certain way. It has since been shut down, and BU was asked to create a replacement. As part of the core infrastructure team, we were tasked with building the backbone of the site and communicating to all other groups about the type of data they could give us and what we, as the CI team, had to accomodate for said type of data by adapting reuseable components. With the given data, we were also tasked with moving it to the database. The site keeps track of all recent boston police activites.

BU Course Search 2.0: Dec 2022-Jan 2023

Tools Utilized: Firebase, React, WebScraping, API

BU's official course search has many unintuitive downsides. It requires constant opening of new tabs, lacking proper quality of life filters, and no review/difficulty tracker for the school. Rather than waiting for the website to get better, I decided to make a better version myself. I scraped the entire course search database, and uploaded it to a seperate firebase database to enable better filters.

ArtGuessr: Dec 2022-Jan 2023

Tools Utilized: Firebase, Google OAuth, Decoupled Architecture, API

Inspired by GeoGuessr, me and three others created a website. We practiced making API calls, decoupling the frontend and backend, and implementing login authentication. The website would display a artwork grabbed from a mueseum API and the user can choose where the artpiece originates from. User's accuracy is tracked by using their google accounts.
